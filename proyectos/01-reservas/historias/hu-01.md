# HU-01: Definir servicios reservables

## Obligatoriedad

Obligatoria

## Historia de usuario

**Como** administrador  
**Quiero** definir servicios con nombre, duración y precio  
**Para** ofrecer citas reservables a los clientes

## Criterios de aceptación

- [ ] Dado el modelo de datos, cuando reviso API o BD, entonces existe entidad Servicio con nombre, duración y precio.
- [ ] Dado un servicio creado vía API o seed, cuando consulto el recurso, entonces devuelve los campos definidos.

## Requiere interfaz web

Parcial

## Procedimiento de verificación (corrector)

1. Revisar modelo/migración Servicio en repo o crear uno vía API.
2. Verificar campos mínimos: nombre, duración (min), precio.
3. Confirmar persistencia en PostgreSQL.
4. Marcar **cumple (2)** si todos los pasos se completan; **parcial (1)** si falta web donde se exige; **no cumple (0)** en caso contrario.
