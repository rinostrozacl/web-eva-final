# HU-07: Evitar doble reserva en mismo slot

## Obligatoriedad

Obligatoria

## Historia de usuario

**Como** sistema  
**Quiero** impedir dos reservas en el mismo slot  
**Para** evitar sobreventa de horarios

## Criterios de aceptación

- [ ] Dado un slot ya reservado, cuando otro usuario intenta reservar el mismo slot, entonces la operación falla con error visible.
- [ ] Dado el conflicto, cuando ocurre en API, entonces responde 409 o 400 con mensaje claro.

## Requiere interfaz web

Sí

## Procedimiento de verificación (corrector)

1. Reservar un slot específico (servicio + fecha + hora).
2. Con otro usuario o sesión, intentar reservar el mismo slot.
3. Verificar mensaje de error en pantalla (no permite confirmar).
4. Opcional: API devuelve 409.
5. Marcar **cumple (2)** si todos los pasos se completan; **parcial (1)** si falta web donde se exige; **no cumple (0)** en caso contrario.
