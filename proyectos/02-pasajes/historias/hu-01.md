# HU-01: Registrar viajes

## Obligatoriedad

Obligatoria

## Historia de usuario

**Como** operador  
**Quiero** registrar viajes con origen, destino, fecha, hora y precio  
**Para** publicar oferta de transporte

## Criterios de aceptación

- [ ] Dado el modelo Viaje, cuando reviso campos, entonces incluye origen, destino, fecha/hora y precio.
- [ ] Dado un viaje creado, cuando consulto API, entonces persiste en BD.

## Requiere interfaz web

Parcial

## Procedimiento de verificación (corrector)

1. Revisar modelo Viaje o crear vía API.
2. Verificar campos obligatorios.
3. Confirmar persistencia.
4. Marcar **cumple (2)** si todos los pasos se completan; **parcial (1)** si falta web donde se exige; **no cumple (0)** en caso contrario.
