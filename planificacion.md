# Planificación — Trabajo grupal MVP (1 semana)

Grupos de **2–5 integrantes**. Una sola entrega al cierre de la semana + presentación en vivo.

## Cronograma sugerido

| Día | Objetivo | Historias orientativas |
|-----|----------|------------------------|
| **D1** | Tema, repo, modelos base | MVP-GEN-01, MVP-GEN-02, MVP-GEN-03, HU-01 |
| **D2** | Auth e inicio dominio | MVP-GEN-04, MVP-GEN-05, HU-02 |
| **D3** | Flujo principal | HU-03, HU-04, HU-05 |
| **D4** | Reglas y vistas | HU-06, HU-07, HU-08 |
| **D5** | Integración front | MVP-GEN-06, pulir UI |
| **D6** | Deploy y pruebas | MVP-GEN-07, pruebas en producción |
| **D7** | Cierre | Matriz, tag `entrega-mvp-grupal`, HU-OPC si hay tiempo |

## Roles sugeridos (flexible)

| Rol | Responsabilidad |
|-----|-----------------|
| Backend | API, modelos, migraciones, reglas de negocio |
| Frontend | Pantallas, consumo API, UX mínima |
| DevOps | `.env`, deploy Railway + front, URLs |
| QA | Matriz de verificación, datos de prueba |
| Todos | Presentación en vivo (cada integrante participa) |

## Entregables

| Elemento | Obligatorio |
|----------|-------------|
| Repositorio Git con `client/` + API | Sí |
| Tag `entrega-mvp-grupal` | Sí |
| 15 historias obligatorias verificables | Sí |
| URLs públicas (API + front) | Sí |
| Matriz de verificación | Sí |
| Presentación demo en vivo (sin PPT) | Sí |
| HU-OPC (opcionales) | No (bonus) |

## Cómo elegir tema

1. Revisar [proyectos.md](proyectos.md).
2. Confirmar con el docente y actualizar tabla de disponibilidad.
3. Un tema por grupo; evitar duplicados en el mismo curso si es posible.

## Integridad académica

- Código propio del grupo.
- No copiar del proyecto individual ni de otros grupos.
- Sin secretos en el repositorio.
